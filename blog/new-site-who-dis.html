I was tired of paying moneys to Wix.com on a subscription basis just for letting me have used their site editor once over 10 years ago. Therefore, I decided to code the website myself. I'm only just now uploading the website to GitHub Pages, and I still need to port over the rest of the content from the previous site to the new site. This site should have all the pieces needed to easily port the content!


This site project was a good excuse to learn CSS (Cascading Style Sheets). On the other hand, JavaScript was similar enough to other programming languages that it didn't really feel like learning it, more like figuring out the documentation and API. Aaaand HTML is still awful as I'm not a fan of tag-based languages, like its counterpart, XML.

Thoughts from making a front-end website from scratch without any prior experience

  • When writing these blogs, doing it in raw HTML sucks. I write them in Obsidian with Markdown and then port them over to HTML using a Markdown to HTML Converter.
  • Similarly, with math equations, HTML has a child format called MathML which sucks to write, which is why Temml - Convert TeX to MathML exists!
  • I really wish there was a way to reuse the Head section in HTML more easily, but I ran into errors trying to encapsulate it.
  • WebStorm as an IDE is OK. Ran into a fun issue where the out-of-the-box WebStorm server includes the project name in the address, so you can't use absolute paths in the HTML files. Worked around it by specifying a relativePath variable in every file that lets file get back to the root of the project to be able to find anything from there. The other solution would have been to run my own server with Apache or something else, which maybe I'll do, but I just wanted to code and figuring out how to use CSS, JavaScript, and HTML and have them all talk to each other was already challenging enough.
  • Making the website mobile compatible feels like making a whole new website, but it does make you either clean up the CSS or hack something in. For example, the navigation bar in the mobile version is always present but offscreen. Not sure how efficient that is, but it works!
  • Adding a dark theme to the site was a fun challenge. My only issue with themes is that I have no idea where to place the theme switcher button! It's next to the logo currently, which looks fine on mobile, but the desktop version makes the logo look a bit sunny...
  • Finding resources to learn how front-end works was challenging at first. At last, I stumbled across Hyperskill Frontend Developer course and their study plan section, which is a great index of everything needed to understand the front-end. Instead of making you go through learning what's a variable, you can skip what you already know and go straight into the unknown!
